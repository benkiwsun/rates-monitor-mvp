# 两周开发排期（MVP）

## Week 1

- Day 1: 需求冻结、指标字典、命名规范
- Day 2: 数据库建模、后端脚手架初始化
- Day 3-4: 数据接入（US/CN/非美主要央行政策利率）
- Day 5: 利差计算、首页快照预聚合、数据质量检查
- Day 6-7: 首页线框搭建（KPI + 主图 + 热力图 + 告警列表）

## Week 2

- Day 8: 告警规则引擎（规则 CRUD + 触发扫描）
- Day 9: 通知通道（邮件 + webhook）
- Day 10: API 完善与鉴权
- Day 11: 传导解读模块（规则模板生成一句话洞察）
- Day 12: 对账与性能优化
- Day 13: 部署与监控
- Day 14: 验收与文档交付

## MVP 验收标准

- 能展示并查询 US/CN 核心利率与中美利差
- 能展示 8 个经济体央行政策利率
- 支持至少 3 条告警规则并能产生事件
- 首页加载 < 2 秒（本地环境）
